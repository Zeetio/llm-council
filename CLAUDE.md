# 絶対遵守
- 英語で思考して日本語で回答
  - 回答はわかりやすく構造的に
  - コード内のコメントは日本語で
- テスト駆動ーすべてのタスク、アウトプットは単体テスト、E2Eテスト（MCP使用）、ビルドテスト、DOCKERビルドテストして終了すること。.envのTEST_EMAIL/TEST_PASSWORDを使用してGOOGLEアカウントにログインする
  - あらゆる処理は適切なスコープでテストが作成、実行されること
  - E2EはChrome DevTools MCP を必ず使用する
    - 使用不可の場合、mcp__playwrightを使用
  - 破壊的変更のデグレは許されない
- 更新の早い情報は必ずインターネット検索で調べて回答
  - AIモデル、フレームワーク、クラウド、ニュース、最新技術はすべて
- コードはクリーンな状態を保つ
  - 新しい機能を追加した場合、不要部を確認して削除や整理する
    - 絶対にディグレしない
  - ファイル、フォルダ整理も適宜行う

## 開発方針
- メンテナビリティを意識した設計、実装
  - 後方互換性は基本入れない（入れない方向性で設計してユーザーに許可を取る）
  - 複雑で局所的な処理は避ける
    - ユーティリティな処理を整備流用する
  - フォルダ、ファイルを整理する
    - 使用されていない
    - 重複している
    - レガシー
  - ユーザーライクな実装
    - ファイル、クラス、関数、コードのコメントをリッチに
- ユーザーに質問を投げるとき、前提や背景をちゃんと説明する
  - 技術的に難しいところを省略しない
  - メリデメを整理する
  - 既存との変更点や、これから起こりうる展開を伝える
